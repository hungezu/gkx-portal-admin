视觉源文件：`/Users/Zhuanz/Desktop/design.md/design.md`。本文件仅记录当前项目的实现摘要；视觉冲突时以源文件为准。

## 视觉令牌

- 主操作蓝：`#165DFF`；悬浮：`#4080FF`；点击：`#0E42D2`
- 页面背景：`#D8E7FC`；工作卡片：`rgba(255,255,255,.90)`
- 主文字：`#1D2129`；次文字：`#4E5969`；三级文字：`#86909C`
- 分隔线：`#E5E6EB`；浅背景：`#F7F8FA`
- 成功：`#00B42A`；警告：`#FF7D00`；危险：`#F53F3F`
- 字体：PingFang SC、PingFang HK、Microsoft YaHei
- 卡片圆角 6px；控件圆角 4px；常规页面卡片不使用阴影

## 布局

- 桌面侧栏 200px，折叠 84px；顶栏 64px。
- 玻璃内容面板距右侧、底部 16px，内部 padding 和模块 gap 均为 16px。
- 内容面板内仅保留一层白色工作卡片。
- `1024px` 以下缩减工具区；`900px` 以下切换移动侧栏；`768px` 以下表格横向滚动。
- 仅工作卡片或表格内容区滚动；侧栏、顶栏和玻璃面板固定。

## 组件

- 主按钮高度 32px、圆角 4px、水平 padding 16px；次按钮白底灰边。
- 输入、选择器、日期控件统一为 32px 高。
- 页面一级及标准内容 Tab 统一使用下划线样式：单项高度36px、宽度随文字 Hug、水平 padding 16px、项间距0，文字为14px/22px；选中项使用 `#165DFF`、500字重，底部显示与单项同宽的2px蓝色下划线。禁止使用填充色块、胶囊、卡片背景或仅与文字同宽的短下划线。该规则同时适用于审核内容、埋点管理、资源管理、权限配置等页面。
- 页面二级 Tab 统一使用分段选择样式：外层 `#F2F3F5`、4px 内边距和 4px 圆角；单项高 32px、水平 padding 16px、项间距 0；选中项白底蓝字且不显示下划线，相邻未选项之间使用 1px 中性分隔线。一级、二级 Tab 视觉层级不得混用。
- 表格表头与数据行均为 40px；单元格左右 padding 为 16px；操作列使用文字链接。
- 表格剩余宽度仅由业务内容列 Fill：按字段语义和内容安全宽度加权分配，名称、标题、长文本、完整 ID 获得更大宽度，状态、枚举、布尔、日期获得较小宽度；禁止把内容列简单等分。
- 多选列固定 52px，操作列固定为最长操作内容 + 12px 操作间距 + 左右各 16px padding；多选列和操作列不参与 Fill，操作列始终位于最右侧，字段内容与操作内容均左对齐，不增加空白占位列。
- 多选框列只在页面明确提供批量操作时出现，并由页面主动开启；没有批量操作的只读表格、仅查看表格或仅有行内操作的表格均不得显示多选框。表格组件默认关闭多选，禁止因使用统一表格组件而自动生成空的52px选择列。
- 详情型长内容（如请求参数、响应详情）不占用常驻数据列；当用户只需按行查看时，在最右侧操作列提供“查看”，点击后通过规范弹窗展示完整内容。
- 普通纯文本不高于 16 个字时完整展示，高于 16 个字时单行省略，并在 hover/focus Tooltip 中显示完整内容。只有用户明确指定的完整展示列（例如流程实例页的流程ID）可按完整内容扩宽，不影响其他列的 16 字规则。
- 分页项为 32px，当前页使用蓝字浅蓝背景，不使用蓝色实心块。
- 弹窗按420 / 640 / 960 / 1280px分级：420px仅用于短确认类，新增、生成、编辑等普通表单类使用640px，复杂配置与穿梭框使用960px，表格或多列权限内容使用1280px；禁止业务页面用旧的420px或720px覆盖上述分级。弹窗为白底、8px圆角并使用高层阴影。标准对话框由固定 Header、可滚动 Body、固定 Footer 三段组成：普通 Header 高56px、左右24px、底部1px分隔线；Body四周24px且仅Body允许纵向滚动；Footer高64px、左右24px、顶部1px分隔线，按钮右对齐且间距12px。取消按钮使用浅灰底，确认按钮使用品牌蓝；提示或说明文案放在 Header 标题正下方，此时Header自适应为至少72px。
- 状态标签使用对应语义浅色背景与文字，不只依赖颜色表达状态。
- 全局轻提示采用页面顶部居中的白底消息条：最小高度 48px，水平 padding 16px，内容间距 12px，1px `#E5E6EB` 边框、4px 圆角和 `0 2px 8px rgba(29,33,41,.12)` 轻阴影；正文为 14px/22px `#1D2129`，右侧保留 24px 可关闭按钮。左侧使用 24px 实心圆形语义图标：信息 `#165DFF`、成功 `#00B42A`、警告 `#FF7D00`、错误 `#F53F3F`。提示背景始终为白色，不使用整条蓝/绿/橙/红色浅底。
- 全局提示默认最多同时堆叠 3 条，间距 8px；信息/成功提示 3 秒自动关闭，警告/错误提示 4.5 秒自动关闭，用户可随时点击关闭。批量导入等复杂流程不得只用轻提示代替页面或弹窗结果状态。
- 弹窗内部告警使用独立 Alert，不复用页面顶部白底轻提示：宽度 Fill、最小高度 40px、padding `9px 12px`、内容间距 12px、4px 圆角、无边框和阴影。信息/成功/警告/错误分别使用 `#E8F3FF` / `#E8FFEA` / `#FFF7E8` / `#FFECE8` 浅色背景，左侧为20px对应语义色实心圆形图标，正文为14px/22px主文字。
- 弹窗 Alert 支持单行正文和“标题 + 描述”两种结构；标题为14px/22px、600字重，标题与描述间距4px。删除确认、Token仅展示一次等风险反馈放在 Modal Body 顶部使用警告或错误 Alert，Alert与后续表单间距16px；普通上下文说明仍放在 Modal Header 标题下方，不得把 Alert、Header 提示和全局轻提示三种组件混用。
- 所有可点击控件必须提供 hover 与 `focus-visible` 状态。

## 数据看板

- 本节规则仅用于数据看板页面及明确包含“指标组 + 图表 + 排行表格”的统计分析页面，不应用于普通列表、表单、详情页或其他业务模块。
- 数据看板模块顶部使用单行标题工具栏：模块标题位于左侧，日期范围等筛选条件位于右侧；窄屏可换行为上下结构。
- 看板内容背景固定为 `#FFFFFF`，不使用灰色模块底。
- 顶部指标组不使用卡片边框、灰底或阴影；相邻指标仅用 `1px` 中性色、`40px` 高且垂直居中的竖线分隔，行末不显示竖线，响应式换列时重新计算或去掉分隔线。
- 指标组下方的同级图表各自使用一层规范线框：白底、`1px solid #E5E6EB`、4px 圆角、16px 内边距、无阴影；禁止再套一层图表区外框。
- 数据看板中的折线点、柱条和漏斗层级必须提供 hover 与键盘 focus 状态。折线/柱图按 Arco Charts 交互：同一横轴位置联动高亮全部系列点，显示竖向指示线，并用一个白底、中性边框、4px 圆角的 Tooltip 同时展示分类及各系列名称和精确值；漏斗高亮当前层级、弱化其余层级，并显示阶段、数值和转化率。
- 图表下方的排行表格继续完整遵守全局表格规范：标题和表格容器与上方图表区域左右边界对齐，排行区外层不再增加卡片 padding；表格使用唯一容器、1px 中性边框、4px 圆角、`#F7F8FA` 表头、40px 行高、单元格左右 16px、单行省略，不得改成无框列表或额外卡片。

## 交互

- 删除操作必须二次确认。
- 审批弹窗同时呈现内容预览、审批进度和意见输入。
- 表格操作以详情、编辑、删除的稳定顺序出现。
- 全局按钮和表格操作链接不使用点状/虚线下划线。禁用操作保持可见时使用禁用灰色与 `not-allowed` 光标，通过 hover/focus Tooltip 解释原因，但不显示虚线且不能触发操作。
- 移动端用遮罩关闭侧栏，表格允许横向滚动。
